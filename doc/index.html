ArCa Payment Gateway

Middleware между Shopify и ArCa (ipay.arca.am). PHP 8.1 + Guzzle 7 + Alpine.js дашборд. Никаких фреймворков — только нативный PHP с PSR-4 автолоадом.

PHP 8.1+ Guzzle 7 vlucas/phpdotenv PDO / MySQL 8 Alpine.js 3 Nginx + PHP-FPM Composer

Архитектура

Routing
public/index.php
Фронт-контроллер. match() по пути URL
Services
ArCa.php / Shopify.php
Guzzle HTTP клиент, типизированные методы API
Persistence
DB.php (PDO)
Singleton, prepared statements, helper-методы
Workers
cron/*.php
CLI скрипты, запускаются системным crontab
💡
Отличие от Node.js версии

PHP версия использует системный cron вместо node-cron, PHP-FPM вместо PM2, и Alpine.js вместо Vanilla JS для дашборда. Бизнес-логика идентична.

Архитектура

Поток платежа (PreAuth mode)

🛒 Shopify Cart→ GET /pay?orderid=→ registerPreAuth.do→ ArCa Form→ GET /result→ Shopify Transaction
Shopify Mark Paid→ POST /webhook/capture→ deposit.do→ ✅ Captured

Sale mode (без PreAuth)

При ARCA_AUTH_MODE=0 используется register.do вместо registerPreAuth.do — деньги списываются сразу, шаг capture не нужен.

Database  Шаг 3

DB.php — PDO Singleton

DB::pdo() создаёт одно соединение на весь запрос (singleton pattern).

php — примеры запросов
// SELECT — один ряд
$tx = DB::fetch(
    'SELECT * FROM transaction WHERE externaltrid = ? LIMIT 1',
    [$arcaOrderId]
);

// SELECT — все ряды
$rows = DB::fetchAll(
    'SELECT * FROM orders WHERE status = ? ORDER BY id DESC',
    [0]
);

// INSERT — возвращает lastInsertId
$id = DB::insert('orders', [
    'status'  => 0,
    'orderid' => '12345',
    'price'   => 43000.00,
]);

// UPDATE
DB::update('transaction',
    ['trstatus' => 'captured'],
    'externaltrid = ?',
    [$arcaOrderId]
);

Route  Шаг 4

GET /pay — инициализация платежа

Файл: src/Routes/Payment.php

Query параметры

ПараметрТипОбязателенОписание
orderidinteger✅Shopify Order ID
langstring—Язык формы ArCa (en, ru, hy)
statusurlURL—Куда вернуть покупателя после результата

Что происходит внутри

  1. Валидация orderid (только цифры)
  2. Shopify::getOrder() — получение данных заказа
  3. Конвертация суммы в AMD × 100 (копейки для ArCa)
  4. DB::insert('orders', ...) — сохранение
  5. ArCa::register() → получаем orderId (UUID) и formUrl
  6. DB::insert('transaction', ...)
  7. header("Location: {$formUrl}") — редирект на форму ArCa
php — конвертация валюты
$amdAmount = match($shopCurrency) {
    'EUR'    => $rawAmount * $cfg['eur_rate'],
    'USD'    => $rawAmount * $cfg['usd_rate'],
    default  => $rawAmount,
};
$amountKopeks = (int) round($amdAmount * 100);
Route  Шаг 5

GET /result — обработка ответа ArCa

Файл: src/Routes/Result.php

ArCa редиректит покупателя сюда после заполнения карточных данных. Эндпоинт вызывает getOrderStatusExtended.do и принимает решение по paymentState:

paymentStateДействиеShopify transaction
APPROVEDСтатус → authorizedkind=authorization, status=success
DEPOSITEDСтатус → capturedkind=sale, status=success
DECLINEDСтатус → declinedkind=authorization, status=failure
otherСтатус → pending—

Дополнительно сохраняет данные карты (PAN masked, holder, approvalCode) в таблицу transaction_details.

Route  Шаг 6

POST /webhook/* — Shopify Events

Файл: src/Routes/Webhook.php

🔐
HMAC Верификация обязательна

Каждый webhook проверяется через hash_equals(base64(hmac_sha256(body, secret)), header) до выполнения любых действий. При несовпадении — 401.

POST /webhook/cancel

Принимает топики orders/cancelled и refunds/create.

  • Если транзакция в статусе authorized → вызывает reverse.do
  • Если refunds/create → вызывает refund.do с суммой рефанда

POST /webhook/capture

Принимает топик orders/paid. Если транзакция authorized → вызывает deposit.do. При неудаче — добавляет задание в таблицу process для Autocapture.

⚡
fastcgi_finish_request()

После отправки 200 ok вызывается fastcgi_finish_request() — PHP-FPM закрывает соединение с Shopify немедленно, а тяжёлая работа (ArCa API вызов) продолжается в фоне.

Worker  Шаг 10

Autopurge — отмена зависших заказов

Файл: src/Workers/Autopurge.php

Находит заказы со статусом status=0 (pending) и транзакцией trstatus=pending, которые старше CANCEL_AFTER_MINUTES минут:

  1. Обновляет transaction.trstatus = 'deleted'
  2. Обновляет orders.status = 4
  3. Вызывает Shopify::cancelOrder()
  4. Вызывает Shopify::deleteOrder()
bash — запуск вручную
php /var/www/arca-rankup-php/cron/autopurge.php
Worker  Шаг 11

Autocapture — повтор deposit.do

Файл: src/Workers/Autocapture.php

Обрабатывает таблицу process — задания, где первый deposit.do не прошёл. До 10 попыток (MAX_RETRIES):

  • Вызывает ArCa::getStatus() — если уже DEPOSITED, просто обновляет статус
  • Если APPROVED → вызывает deposit.do
  • При успехе — удаляет запись из process
  • При неудаче — увеличивает retry
bash — запуск вручную
php /var/www/arca-rankup-php/cron/autocapture.php
Cron

Настройка Cron

В отличие от Node.js версии, здесь нет встроенного планировщика — используется системный crontab:

bash — crontab -e
# Autopurge — каждые 5 минут
*/5 * * * * php /var/www/arca-rankup-php/cron/autopurge.php  >> /var/log/arca-cron.log 2>&1

# Autocapture — каждые 2 минуты
*/2 * * * * php /var/www/arca-rankup-php/cron/autocapture.php >> /var/log/arca-cron.log 2>&1
⚠️
PHP Binary Path

Убедитесь что используете правильный бинарник: which php или явно /usr/bin/php8.1. На серверах с несколькими версиями PHP php по умолчанию может быть 7.x.

Установка PHP 8.1 + расширения

bash — Ubuntu 22.04
sudo add-apt-repository ppa:ondrej/php -y
sudo apt update
sudo apt install -y php8.1 php8.1-fpm php8.1-mysql php8.1-curl \
                   php8.1-mbstring php8.1-xml php8.1-zip

# Проверка
php8.1 -v
systemctl status php8.1-fpm

Nginx конфигурация

/etc/nginx/sites-available/arca-gateway
server {
    listen 443 ssl http2;
    server_name checkout.your-domain.com;

    ssl_certificate     /etc/letsencrypt/live/checkout.your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/checkout.your-domain.com/privkey.pem;

    root  /var/www/arca-rankup-php/public;
    index index.php;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location ~ \.php$ {
        include        fastcgi_params;
        fastcgi_pass   unix:/run/php/php8.1-fpm.sock;
        fastcgi_index  index.php;
        fastcgi_param  SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
    }

    # Запрет доступа к исходникам
    location ~ /\.(env|git) { deny all; }
    location ^~ /src/       { deny all; }
    location ^~ /cron/      { deny all; }
    location ^~ /logs/      { deny all; }
}

Docker (опционально)

Если хотите запустить в контейнере, добавьте Dockerfile на основе php:8.1-fpm-alpine:

Dockerfile
FROM php:8.1-fpm-alpine

RUN apk add --no-cache curl \
 && docker-php-ext-install pdo_mysql \
 && curl -sS https://getcomposer.org/installer | php -- --install-dir=/usr/local/bin --filename=composer

WORKDIR /app
COPY composer*.json ./
RUN composer install --no-dev --optimize-autoloader --no-scripts

COPY . .
RUN mkdir -p logs && chown -R www-data:www-data logs

USER www-data
EXPOSE 9000
CMD ["php-fpm"]
ℹ️
PHP-FPM + Nginx в Docker

PHP-FPM не принимает HTTP напрямую — нужен отдельный Nginx контейнер в docker-compose.yml. FastCGI соединяется по сокету или fastcgi_pass app:9000.
